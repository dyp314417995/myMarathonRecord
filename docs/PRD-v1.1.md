# 我的跑马记录 — PRD 产品需求文档 v1.1

> 版本：V1.1 | 状态：规划中 | 模块：积分管理

---

## 1. 概述

在 V1.0 用户管理基础上，新增积分体系。用户可通过拉新、参赛、自媒体发帖等行为赚取积分，积分可抵扣商品购买金额（纯记录，暂不接支付）。

---

## 2. 积分规则

### 2.1 赚积分

| 分类 | 积分 | 频率上限 | 申请方式 |
|------|:---:|:---:|------|
| 拉新 | 3 分/人 | 不限 | 用户提交申请 |
| 团服参赛 | 5 分/次 | 不限 | 用户提交申请 |
| 自媒体 | 3 分/次 | **每月 4 次** | 用户提交申请 |
| 集体活动 | 3 分/次 | 不限 | **管理员手动录入** |

- 自媒体要求：带「九州战马联盟」话题并 @九州战马联盟
- 每个分类独立计数，互不影响

### 2.2 用积分

| 规则 | 说明 |
|------|------|
| 兑换比例 | 1 分 = 1 元 |
| 最大折扣 | 商品原价的 85 折 |
| 适用范围 | 迈胜、必迈等产品 |
| 不适用 | 特价活动、团购活动 |
| 操作方式 | 管理员手动录入消耗（V1.1 不做支付对接） |

### 2.3 注册赠送

新用户注册完成后自动获得 **50 积分**。

### 2.4 积分有效期

- 自然年有效：获得日期 + 365 天
- 到期自动失效，余额扣减
- 过期提醒：到期前 **30 天、7 天、1 天** 各推送一次

## 4. 数据库设计（新增）

### 4.1 points_rules — 积分规则

| 字段 | 类型 | 示例 |
|------|------|------|
| category | String | `自媒体` |
| name | String | `自媒体发帖` |
| points | Number | `3` |

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | String | 所属用户 |
| type | String | `earn` / `use` / `expire` |
| category | String | `拉新` / `团服参赛` / `自媒体` / `集体活动` / `注册赠送` / `消耗` |
| points | Number | 正数=赚，负数=花 |
| balance | Number | 变动后余额（快照） |
| description | String | 用户填写的说明 |
| images | Array | 上传的图片 cloud fileID |
| monthlyIndex | Number | 本月第几次 |
| earnDate | Date | 积分获得日期 |
| expireDate | Date | earnDate + 365 天 |
| reviewerId | String | 审批人 |
| status | String | `pending` / `approved` / `rejected` / `expired` |
| createTime | Date | 创建时间 |

## 7. 关键逻辑

### 7.1 本月次数计算

```js
countRecords(userId, category, 当月1号, 当月最后一天, status='approved')
```

### 7.2 频率校验

提交申请时：
```
本月次数 >= monthlyLimit → 拒绝提交
否则 → 允许
```

### 7.3 余额计算

实时计算为所有 `approved` 状态的 `earn` 积分之和减去 `use` 积分。`balance` 快照字段用于快速显示，定时任务同步校正。

### 7.4 过期处理

云函数 `pointsExpireCheck` 每天执行：
1. 查询 `expireDate <= 当天 && status='approved'` 的记录
2. 更新 `status='expired'`
3. 检查未来 30/7/1 天即将过期的积分，推送订阅消息提醒
