# 我的跑马记录 — PRD 产品需求文档 v1.1

> 版本：V1.1 | 状态：已开发 | 模块：积分管理 + V1.0 修复优化

---

## 1. 概述

在 V1.0 用户管理基础上，新增积分体系。用户可通过拉新、参赛、自媒体发帖等行为赚取积分，积分可抵扣商品购买金额（纯记录，暂不接支付）。

---

## 2. 积分规则

### 2.1 赚积分

| 分类 | 积分 | 频率上限 | 申请方式 |
|------|:---:|:---:|------|
| 拉新 | 3 分/人 | 不限 | 用户提交申请 |
| 团服参赛 | 5 分/次 | 不限 | 用户提交申请 |
| 自媒体 | 3 分/次 | **每月 4 次** | 用户提交申请 |
| 集体活动 | 3 分/次 | 不限 | **管理员手动录入** |

- 自媒体要求：带话题并 @九州战马联盟
- 每个分类独立计数，互不影响

### 2.2 用积分

| 规则 | 说明 |
|------|------|
| 兑换比例 | 1 分 = 1 元 |
| 最大折扣 | 商品原价的 85 折 |
| 适用范围 | 迈胜、必迈等产品 |
| 不适用 | 特价活动、团购活动 |
| 操作方式 | 管理员手动录入消耗（V1.1 不做支付对接） |

### 2.3 注册赠送

新用户注册自动获得 **50 积分**。V1.0 老用户首次登录自动补发。

### 2.4 积分有效期

- 自然年有效：获得日期 + 365 天
- 到期自动失效，余额扣减
- 过期提醒：到期前 30 天、7 天、1 天各推送一次（需订阅消息模板）

## 3. 功能模块

### 3.1 用户端

| 功能 | 说明 |
|------|------|
| 积分首页 | 余额 + 即将过期提醒 + 收支记录 + 积分规则（可折叠查看） |
| 提交申请 | 选类型、上传图片（必填）、填写说明（选填）、显示本月次数 |
| 频率限制 | 达到月上限时不可提交 |
| 积分规则 | 可折叠展示，只读 |

### 3.2 管理员端

| 功能 | 超管 | 管理员 | 说明 |
|------|:---:|:---:|------|
| 积分规则配置 | ✅ | ❌ | 启用/禁用规则 |
| 审批积分申请 | ✅ | ✅ | |
| 手动录入积分 | ✅ | ✅ | 正=加分，负=扣分 |
| 用户列表搜索排序 | ✅ | ✅ | 按姓名/城市/群搜索，按时间/姓名/10K/半马/全马排序（正序/倒序） |
| 用户详情弹窗 | ✅ | ✅ | 点用户查看城市/群/PB/注册时间/积分，不显示手机号 |

### 3.3 审批增强

| 功能 | 说明 |
|------|------|
| 用户详情弹窗 | 审批时点用户头像查看城市和 PB 成绩 |
| 隐私保护 | 详情弹窗不显示手机号 |

## 4. 数据库设计（新增）

### 4.1 points_rules

| 字段 | 类型 | 说明 |
|------|------|------|
| category | String | `拉新` / `团服参赛` / `自媒体` / `集体活动` |
| name | String | 如 "带话题并@九州战马联盟" |
| points | Number | 每次积分值 |
| monthlyLimit | Number/null | 月上限，null=不限 |
| status | String | `active` / `disabled` |

### 4.2 points_records

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | String | 所属用户 |
| type | String | `earn` / `use` / `expire` |
| category | String | 积分分类 |
| points | Number | 正数=赚，负数=花 |
| description | String | 说明 |
| images | Array | cloud fileID |
| monthlyIndex | Number | 本月第几次 |
| earnDate | Date | 获得日期 |
| expireDate | Date | 失效日期 |
| reviewerId | String | 审批人 |
| status | String | `pending` / `approved` / `rejected` / `expired` |
| createTime | Date | 创建时间 |
