# 我的跑马记录 — PRD 产品需求文档 v1.1

> 版本：V1.1 | 状态：已开发 | 模块：积分管理 + V1.0 修复优化

---

## 1. 概述

在 V1.0 用户管理基础上，新增积分体系。用户可通过拉新、参赛、自媒体发帖等行为赚取积分，积分可抵扣商品购买金额（纯记录，暂不接支付）。

---

## 2. 积分规则

### 2.1 赚积分

| 分类 | 积分 | 频率上限 | 申请方式 |
|------|:---:|:---:|------|
| 拉新 | 3 分/人 | 不限 | 用户提交申请 |
| 团服参赛 | 5 分/次 | 不限 | 用户提交申请 |
| 自媒体 | 3 分/次 | **每月 4 次** | 用户提交申请 |
| 集体活动 | 3 分/次 | 不限 | **管理员手动录入** |

- 自媒体要求：带话题并 @九州战马联盟
- 每个分类独立计数，互不影响

### 2.2 用积分

| 规则 | 说明 |
|------|------|
| 兑换比例 | 1 分 = 1 元 |
| 最大折扣 | 商品原价的 85 折 |
| 适用范围 | 迈胜、必迈等产品 |
| 不适用 | 特价活动、团购活动 |
| 操作方式 | 管理员手动录入消耗（V1.1 不做支付对接） |

### 2.3 注册赠送

新用户注册自动获得 **50 积分**。V1.0 老用户首次登录自动补发。

### 2.4 积分有效期

- 自然年有效：获得日期 + 365 天
- 到期自动失效，余额扣减
- 过期提醒：到期前 30 天、7 天、1 天各推送一次（需订阅消息模板）

## 9. V1.1 功能优化

### 9.1 取消入群审批

- 用户选择群组直接加入，不再产生 pending 状态
- 不再使用 `join_requests` 表创建审批记录
- 用户选群即生效，注册按钮显示"完成注册"

### 9.2 群介绍与备注

| 字段 | 说明 |
|------|------|
| description | 群介绍（管理员编辑） |
| remark | 备注，默认"群已满，请联系管理员邀请加入" |

- 管理员在群组管理 → 编辑弹窗中可填写
- 未上传二维码时，用户端显示 remark 提示

### 9.3 跑群列表页

- 新增 `pages/groups/index`
- 用户首页 → 📋 跑群列表
- 展示：群名称、群介绍、二维码（可点击放大识别）、无码时显示备注

### 9.4 群成员数

- 加入/退出/移出群组时更新 `memberCount`
- 使用 `_.inc(1)` / `_.inc(-1)` 原子操作

### 9.5 删除用户

- 管理员可删除用户（`users` 集合 `remove`）
- 删除后扣减对应群成员数
- 用户可重新注册
- 超管不可被删除

### 9.6 解除管理员

- 用户列表 → 超管可点「解除管理员」
- 用户角色变回 `user`，admin 记录标记 revoked
- 普通管理员不可操作（仅超管可见）

### 9.7 权限细化

| 操作 | 超管 | 管理员 |
|------|:---:|:---:|
| 删除用户 | ✅ | ✅ |
| 解除管理员 | ✅ | ❌ |
| 积分规则启用/禁用 | ✅ | ❌ |
| 积分录入/扣减 | ✅ | ✅ |
| 积分审批 | ✅ | ✅ |

### 9.8 移除入群审批

- 注册时 status 固定为 `approved`，不创建 join_requests
- 首页移除「审批加入」入口和待审批统计
- 管理员权限从"审批加入"改为"积分审批"

### 9.9 管理员去重

- 添加管理员时自动撤销该用户已有记录
- 列表同用户只显示最新一条
- 删除用户时同步清理管理员记录

### 9.10 管理员积分申请

- 管理员提交申请自动生效（status=approved），无需审批
- 管理员可见「集体活动」分类，普通用户不可见
- 非管理员提交仍走审批流程

### 9.11 其他

- 手机号改为纯手动输入，去除微信获取按钮
- 已被删除用户重新打开小程序自动清缓存跳注册页
- 积分明细时间格式化为可读字符串
