# 部署指南

## 前置条件

- 微信小程序已注册（AppID）
- 开通云开发（微信云）

## 一、云开发环境

### 1.1 数据库集合

| 集合名 | 说明 |
|--------|------|
| `users` | 用户 |
| `groups` | 跑群 |
| `admins` | 管理员 |
| `points_records` | 积分记录 |
| `points_rules` | 积分规则 |
| `join_requests` | 入群申请（历史） |
| `race_records` | 跑马记录（v1.4 新增） |

### 1.2 云函数

| 函数名 | 环境变量 | 说明 |
|--------|----------|------|
| `login` | 无 | 获取 openid |
| `getImageUrls` | 无 | 管理员权限图片转临时链接 |
| `fixPoints` | 无 | 一次性修复缺失积分字段 |
| `aiCoach` | `QWEN_KEY` | AI 教练（千问） |

### 1.3 数据库权限

所有集合设置为「仅创建者可读写」。

## 二、云函数部署

右键 `cloudfunctions/xxx` → **上传并部署：云端安装依赖**。

### aiCoach

1. 上传后进入云开发控制台 → 云函数 → aiCoach
2. 环境变量添加 `QWEN_KEY`（千问 API Key）
3. 超时设为 10 秒
4. 如需图片分析（qwen-vl），需保留 `wx-server-sdk` 依赖

## 三、小程序发布

1. 开发者工具 → 上传代码 → 填写版本号
2. 微信公众平台 → 版本管理 → 提交审核
3. 审核通过 → 提交发布（全量发布）

## 四、v1.4 部署步骤

1. 云开发控制台 → 数据库 → 新建集合 `race_records`
2. 权限设置为「仅创建者可读写」
3. 上传 `aiCoach` 云函数（如有修改）
4. 编译 → 上传代码 → 提交审核

## 五、常见问题

**Q: 图片/头像不显示？**
A: 检查 `getImageUrls` 云函数是否已部署。

**Q: AI 教练返回错误？**
A: 检查云函数环境变量 `QWEN_KEY`，确认千问 API Key 有效。

**Q: 搜索后头像变空白？**
A: v1.3 已修复，超过 50 个 cloud:// 头像需分批次转换。
