## 四、积分管理（管理端）

| # | 场景 | 操作 | 预期结果 | 结果 |

### 4.2 审批积分

| # | 场景 | 操作 | 预期结果 | 结果 |

| 4.2.1 | 通过申请 | 审批 Tab → 点「通过」 | 用户积分增加，记录状态变为 approved | |
| 4.2.2 | 拒绝申请 | 审批 Tab → 点「拒绝」 | 记录状态变为 rejected | |

| 4.2.4 | 超管/管理员都可见 | 两种角色分别打开 | 审批列表正常 | |

### 4.3 录入/扣减

| # | 场景 | 操作 | 预期结果 | 结果 |
|---|------|------|------|:---:|
| 4.3.1 | 加分 | 选用户 → 输入 30 → 填写原因 → 确认 | 该用户积分 +30 | |
| 4.3.2 | 扣分 | 选用户 → 输入 -30 → 填写原因 → 确认 | 该用户积分 -30 | |
| 4.3.3 | 余额不足 | 余额 10 → 扣 30 | 提示"余额不足" | |
| 4.3.4 | 输入过滤 | 输入中文或特殊字符 | 自动过滤，只保留数字和 - | |
| 4.3.5 | 非管理员不可操作 | 普通用户进此页 | 显示"仅管理员可操作" | |

### 4.4 管理员权限
