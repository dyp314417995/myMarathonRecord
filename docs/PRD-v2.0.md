# 赛事日历 — PRD 产品需求文档 v2.0

> 版本：V2.0 | 状态：已完成 | 模块：赛事日历 + 赛事体验评分 + 倒计时通知

---

## 1. 概述

在小工具区新增「赛事日历」功能，展示马拉松赛事，支持查看、标记、倒计时、评分与分享。新增「赛事体验」评分体系，用户可对赛事进行多维度打分和标签推荐，首次评价自动获得 10 积分（无需审核），每人每赛事限评一次但可修改。管理员可查看/删除所有评价。

---

## 2. 页面架构

```
pages/tools/calendar/index       → 赛事日历主页（所有赛事 / 我的赛事）
pages/tools/calendar/detail      → 赛事详情页（含评分入口）
pages/tools/calendar/review      → 赛事评分页（多维度打分）
pages/tools/calendar/review-list → 赛事评分详情（评分明细 + 标签排行）
```

**首页入口**：小工具区 → 赛事日历（替换现有的 disabled 占位）

## 3. 赛事日历

### 3.1 数据来源

- **管理员手动录入**（赛事太多，精选录入更可控）
- 支持后续接入田协接口自动同步（预留）
- 存储集合：`race_events`

### 3.1.1 管理员录入页面

独立页面 `pages/admin/races/races`，从首页「管理」面板进入（与群组管理、积分管理并列）。表单字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 赛事名称 | text | 必填 |
| 省份/城市 | picker | 选填 |
| 赛事类型 | 全马/半马/10K/越野 | 必填 |
| 赛事等级 | A/B/C | 选填 |
| 赛事标牌 | 白金标/金标/精英标/普通标 | 选填 |
| 距离(km) | text | 越野必填 |
| 爬升(m) | text | 越野选填 |
| 官网链接 | text | 选填 |
| 赛事规模 | text | 选填 |
| 报名费用 | text | 选填 |
| 报名机制 | 抽签/先到先得 | 选填 |
| 付费方式 | 先缴费/中签后缴费 | 选填 |
| **时间节点** | date + time | 鸣枪开跑必填，其余选填 |
| 海报图片 | image | 选填 |

支持列表管理（增删改），默认按日期倒序。每条时间节点精确到小时（时间选择器），默认 12:00（鸣枪开跑默认 07:00）。

**时间节点顺序**：报名开启 → 报名截止 → 退费截止 → 出签时间 → 缴费截止 → 候补时间 → 二抽出签 → 鸣枪开跑

### 3.1.2 状态自动更新

云函数定时器每天凌晨执行：
- `date < 今天` 且 `status = 'upcoming'` → 自动改为 `finished`

### 3.2 赛事字段

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 赛事名称 |
| date | Date | 举办日期 |
| city | String | 所在城市 |
| province | String | 所在省份 |
| raceType | String | 全马/半马/10K/越野 |
| raceLevel | String | A/B/C 等级 |
| distance | String | 距离(km)，越野必填 |
| elevation | String | 爬升(m)，越野必填 |
| certs | Object | 认证：{itra, utmb, utmbws} |
| status | String | upcoming / finished |
| website | String | 官网链接（选填） |
| poster | String | 海报图 cloud fileID |
| tags | Array | 赛事标签（手动维护） |
| createTime | Date | 创建时间 |

### 3.3 日历主页

**默认视图**：当天日期起近三个月赛事，默认打开「我的赛事」Tab。

**时间筛选**：顶部日期区间选择器，支持自定义起止日期。

**Tab 切换**：

| Tab | 说明 |
|-----|------|
| 我的赛事 | 当前用户标记过的赛事（默认） |
| 所有赛事 | 全部符合条件的赛事列表 |
| 我的评价 | 当前用户评价过的赛事，可编辑/删除 |

**排序**（下拉选择）：

| 排序方式 | 说明 |
|----------|------|
| 综合评分 | 按赛事综合平均分降序 |
| 赛道难度 | 按难度评分降序 |
| 赛道氛围 | 按氛围评分降序 |
| 补给 | 按补给评分降序 |
| 交通 | 按交通评分降序 |
| 日期 | 按赛事日期升序/降序 |

**标签筛选**：支持按赛事标签过滤（如"PB赛道"、"新手友好"等）。

### 3.4 赛事卡片

每个赛事卡片展示：

- 赛事名称、日期
- **倒计时**：精确到小时 —— 距离最近时间节点 < 1h 显示"即将"；1~23h 显示"距XX X小时"；≥24h 显示"距XX X天"
- 城市、赛事等级标签
- 综合评分（星星 + 分数 + 评分人数）
- 报名未开启的赛事显示 🟠「未开启报名」标签
- 「标记我的赛事」按钮（已标记显示状态标签）
- 点击卡片进入详情页

### 3.5 标记我的赛事

标记赛事时弹出选择框：

| 状态选项 | 说明 | 与跑马记录同步 |
|----------|------|:---:|
| 计划报名 | 打算报名 | |
| 已报名 | 已完成报名 | ✅ |
| 已中签 | 抽签中签 | ✅ |
| 未中签 | 抽签未中签 | |
| 已完赛 | 已完赛 | ✅ |
| 未完赛(DNF) | 未完赛 | ✅ |
| 弃赛(DNS) | 弃赛 | ✅ |

**🔔 倒计时通知**：标记时可开启倒计时提醒，将在每个时间节点的**前一天上午 9:00** 推送订阅消息，告知具体时间（如"明天 08:00 鸣枪开跑！"）。

**成绩同步**：
- 已完赛可填写完赛成绩，自动同步到「我的跑马记录」
- 完赛且成绩优于当前 PB → 自动更新个人资料 PB

## 4. 赛事体验评分

### 4.1 评分入口

- 赛事详情页底部 → 「参与评分」按钮（首次）/ 「修改评价」按钮（已有评价）
- 每位用户对每个赛事只能提交一次评价，提交后可修改
- 首次评价获得 10 积分，修改不重复发放
- 详情页展示「📝 我的评价」，可查看各维度分数和标签，支持删除
- **删除评价**：自动扣减 10 积分，删除后可重新评价再次获得 10 积分
- 评分页也支持删除（仅已有评价时显示删除按钮）

### 4.2 评分维度（1-10 分）

| 维度 | 说明 |
|------|------|
| 赛道难度 | 1=轻松 10=极难 |
| 赛道氛围 | 观众热情、加油氛围 |
| 补给 | 补给站数量与质量 |
| 交通 | 到达赛场的便利性 |
| 赛道风景 | 沿途风景质量 |
| 组织水平 | 赛事组织、引导、存包等 |
| 完赛奖牌 | 奖牌设计与质量 |
| 性价比 | 报名费与体验的匹配度 |

**综合评分**：所有维度加权平均，按评分人数取均值。

### 4.3 额外信息

用户评分时补充：

| 字段 | 说明 |
|------|------|
| 爬升(m) | 手动填写实际爬升 |
| 使用装备 | 手动填写（跑鞋、手表等） |

### 4.4 评分数据存储

集合：`race_reviews`

| 字段 | 类型 | 说明 |
|------|------|------|
| eventId | String | 赛事 ID |
| userId | String | 评分用户 |
| scores | Object | 各维度评分 {difficulty:8, atmosphere:7, ...} |
| elevation | String | 爬升(m) |
| equipment | String | 使用装备 |
| tags | Array | 用户打的标签 |
| status | String | approved（提交即生效） |
| createTime | Date | 创建时间 |
| updateTime | Date | 修改时间 |

### 4.5 综合评分计算

- 所有评分提交即生效，实时计入统计
- 管理员可在赛事详情页「🛡 管理评价」面板查看/删除恶意评价
- 综合分 = 各维度平均分的平均值
- 赛事卡片展示：综合分 + 参评人数

## 5. 标签系统

### 5.1 旅行/美食/住宿标签

用户在评分时可为赛事打标签（多选）：

| 分类 | 标签示例 |
|------|----------|
| 旅行 | 值得一游、顺便旅游 |
| 美食 | 当地美食多、补给好吃 |
| 住宿 | 住宿方便、性价比高 |
| 赛道 | PB赛道、新手友好、坡多、风景好 |
| 综合 | 推荐参赛、不推荐 |

### 5.2 标签展示

赛事详情 → 标签区：

- 按被标记次数降序排列
- 每个标签显示「标签名 ×被标记次数」
- 排名前 10 的标签高亮展示

### 5.3 标签数据存储

在 `race_events` 中维护标签统计（避免每次聚合）：

| 字段 | 类型 | 说明 |
|------|------|------|
| tagStats | Object | { "PB赛道": 15, "美食多": 8, ... } |

每次评分提交后实时更新 `tagStats`。

## 6. 倒计时通知

### 6.1 通知机制

- 用户在标记赛事时开启 🔔 倒计时提醒
- 开启时触发 `wx.requestSubscribeMessage` 获取订阅授权
- 云函数 `checkRaceNotify` 每天 **9:00** 定时扫描
- 检测所有开启通知的赛事，找出**明天到期**的时间节点

### 6.2 通知话术

每个时间节点有独立话术，包含具体时间：

| 节点 | 话术示例 |
|------|------|
| 报名开启 | 明天10:00 报名开启！准备好报名资料，定好闹钟别错过 |
| 报名截止 | 明天12:00 报名截止！还没报名的跑友抓紧最后机会 |
| 退费截止 | 明天12:00 退费截止，需要退费请尽快操作 |
| 出签时间 | 明天12:00 公布中签结果，祝你好运！🍀 |
| 缴费截止 | 明天12:00 缴费截止，中签后请及时缴费锁定名额 |
| 候补时间 | 明天12:00 开启候补，还有机会！别放弃 |
| 二抽出签 | 明天12:00 二轮抽签出结果，再试试手气！ |
| 鸣枪开跑 | 明天07:00 鸣枪开跑！检查装备，早睡早起，加油！🏃 |

### 6.3 订阅消息模板

- 模板 ID：`xepY9QmUT4YPXUt7mLhQrtlVBcN01eGkk-NqH0Av5Ew`
- 字段：`thing1`（提醒类型）、`thing2`（赛事名+内容）、`thing3`（状态说明）
- 点击消息跳转：赛事详情页

## 7. 分享功能

### 7.1 赛事分享卡片

- 点击赛事详情页右上角「…」分享
- 生成小程序分享卡片：赛事名称 + 日期 + 城市 + 综合评分
- 路径携带 `eventId`，扫码直接进入该赛事详情页

### 7.2 新用户引导

- 未注册用户扫码 → 跳转注册页
- 注册完成后 → 自动跳转到扫码的赛事详情页（通过 URL 参数传递 `eventId`）

## 8. 积分奖励

### 8.1 评分奖励

- 用户首次提交赛事评分 → 自动获得 **10 积分**（无需审核）
- 修改评价不重复发放积分
- 积分规则展示在规则列表中但不出现在申请页（与集体活动类似，由系统自动发放）

### 8.2 积分流程

```
用户提交评分 → status=approved → 10积分秒到账 + 评分实时计入统计
管理员发现恶意 → 详情页「🛡 管理评价」删除记录
```
