# V2.0 赛事数据采集方案

## 决策：手动录入为主

赛事太多、用户只关心热门赛事，自动爬取反而数据冗余。采用**管理员手动录入**，精选可控。

---

## 实现

### 管理员赛事管理页

在积分管理后台新增「赛事管理」Tab（`pages/points/admin` → 加一个 tab），表单包含：

| 字段 | 说明 |
|------|------|
| 赛事名称 | 必填 |
| 举办日期 | 必填 |
| 省份/城市 | 选填 |
| 赛事类型 | 全马/半马/10K/越野 |
| 赛事等级 | A/B/C |
| 官网 | 选填 |
| 海报图 | 选填 |

支持增删改查，列表按日期倒序。

### 状态自动更新

云函数定时器，每天凌晨跑一次：
- `race_events` 中 `date < 今天 && status === 'upcoming'` → 改为 `finished`
