# 我的跑马记录 — 部署文档 v1.1

> 适用版本：V1.1 积分管理 | 基于 V1.0 已部署环境

---

## 新增内容概览

| 类型 | 名称 | 说明 |
|------|------|------|
| 页面 | `pages/points/index` | 积分首页 |
| 页面 | `pages/points/apply` | 提交积分申请 |
| 页面 | `pages/points/admin` | 积分管理后台 |
| 工具 | `utils/points.js` | 积分操作封装 |
| 云函数 | `pointsExpireCheck` | 过期处理 + 提醒 |
| 集合 | `points_rules` | 积分规则 |
| 集合 | `points_records` | 积分流水 |

---

## 1. 数据库新建集合

云开发控制台 → 数据库 → 添加集合：

| 集合名 | 权限（先用） |
|--------|:---:|
| `points_rules` | read: true, write: true |
| `points_records` | read: true, write: true |

---

## 2. 部署云函数

```
右键 cloudfunctions/pointsExpireCheck → 「上传并部署：云端安装依赖」
```

部署后在云开发控制台 → 云函数 → `pointsExpireCheck` → 确认状态为「部署成功」。

> ⚠️ 定时触发器会自动创建，无需手动配置。

---

## 3. 订阅消息模板（可选）

如果想开启积分过期提醒推送：

1. 微信公众平台 → 功能 → 订阅消息
2. 申请「积分到期提醒」类模板
3. 将模板 ID 填入 `cloudfunctions/pointsExpireCheck/index.js` 的 `templateId` 字段

> 不配置也不影响核心功能，只是不推送提醒。

---

## 4. 初始化积分规则

首次打开「积分管理 → 规则」会自动写入 4 条默认规则：

| 分类 | 积分 | 月上限 |
|------|:---:|:---:|
| 拉新 | 3 | 不限 |
| 团服参赛 | 5 | 不限 |
| 自媒体 | 3 | 4次 |
| 集体活动 | 3 | 不限 |
