# PRD 产品需求文档 v2.2 — 活动广场

> 版本：V2.2 | 状态：设计中 | 模块：活动管理 + 活动广场

---

## 1. 概述

管理员可发布线下/线上活动，用户报名参与。活动完成后自动获得积分。

---

## 2. 管理员端 — 活动管理

### 2.1 入口
首页 → 管理 → 活动管理

### 2.2 发布活动

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| 活动名称 | 文本 | ✅ | |
| 活动时间 | 日期+时间 | ✅ | 支持多天（开始-结束） |
| 活动地点 | 文本 | ✅ | |
| 费用 | 文本 | | 如"免费""AA制 50元" |
| 报名截止时间 | 日期+时间 | ✅ | |
| 人数上限 | 数字 | | 不填=不限 |
| 活动图片 | 图片 | | 最多 3 张 |
| 活动说明 | 富文本 | | 图文混排 |

### 2.3 自定义扩展表单

管理员可自行添加额外报名字段：

| 字段属性 | 选项 |
|----------|------|
| 类型 | 文本 / 单选 / 多选 / 数字 |
| 标题 | 如"衣服尺码""是否团餐" |
| 必填 | 是/否 |
| 选项 | 单选/多选时填写，逗号分隔 |

示例：
- 衣服尺码（单选）：S, M, L, XL, XXL
- 是否团餐（单选）：是, 否
- 备注需求（文本）

### 2.4 活动列表

管理端活动列表展示：
- 活动名称、时间、地点
- 报名人数 / 上限
- 操作：编辑、查看报名、取消活动、删除
- 取消后所有报名自动取消，不发放积分

### 2.5 查看报名详情

点击"查看报名"弹窗/跳转：
- 报名人员列表（头像、昵称、手机号）
- 自定义字段的填写内容（如表）
- 支持导出（可选）

---

## 3. 用户端 — 活动广场

### 3.1 入口
首页 → 小工具 → 活动广场

### 3.2 Tab 结构

| Tab | 说明 |
|-----|------|
| 所有活动 | 展示未过期的全部活动，按时间从近到远 |
| 我的活动 | 已报名的活动，按时间从近到远 |

### 3.3 活动卡片

每个卡片展示：
- 活动名称
- 活动时间（开始时间）
- 地点
- 报名人数 / 上限
- 费用（若有）

### 3.4 活动详情

点击卡片进入详情页：
- 活动全部信息（时间/地点/费用/说明/图片）
- **报名按钮**（未截止且未满员时）
- 自定义表单字段（报名时填写）

### 3.5 报名流程

1. 点击"立即报名"
2. 填写自定义字段
3. 确认提交
4. 该活动进入"我的活动"

### 3.6 取消报名

"我的活动"中可取消报名，截止时间前可操作。

### 3.7 积分奖励（自动）

参考「集体活动」积分规则。活动结束后：

- 管理员可选择"活动已完成"
- 系统自动给所有报名用户 +3 积分
- 积分规则走现有 `集体活动` 分类（`points: 3`）

## 4. 数据模型

### 4.1 `activities` 集合

```js
{
  _id: String,
  name: String,           // 活动名称
  timeStart: Date,        // 开始时间
  timeEnd: Date,          // 结束时间
  location: String,       // 地点
  fee: String,            // 费用
  deadline: Date,         // 报名截止时间
  maxPeople: Number,      // 人数上限（null=不限）
  images: [String],       // 活动图片 fileID
  description: String,    // 活动说明
  customFields: [{        // 自定义字段
    key: String,          // 字段标识
    label: String,        // 字段标题
    type: 'text'|'radio'|'checkbox'|'number',
    required: Boolean,
    options: [String],    // 单选/多选的选项
  }],
  status: 'active'|'finished'|'cancelled',
  creatorId: String,      // 管理员 openId
  createTime: Date,
}
```

### 4.2 `activity_registrations` 集合

```js
{
  _id: String,
  activityId: String,     // 关联活动
  userId: String,         // 用户 _id
  userName: String,       // 冗余：用户昵称
  userPhone: String,      // 冗余：手机号
  customValues: {         // 自定义字段值
    "size": "L",
    "dinner": "是",
  },
  status: 'active'|'cancelled',
  createTime: Date,
}
```

## 6. 权限

| 操作 | 普通用户 | 管理员 |
|------|:---:|:---:|
| 查看所有活动 | ✅ | ✅ |
| 报名/取消报名 | ✅ | ✅ |
| 发布/编辑/取消/删除活动 | ❌ | ✅ |
| 查看报名详情 | ❌ | ✅ |
| 标记活动完成 | ❌ | ✅ |

---

## 7. 积分联动

活动标记完成后，遍历 `activity_registrations` 中 `status='active'` 的记录，调用 `pointsUtil.addRecord()`：

```js
{
  userId,
  type: 'earn',
  category: '集体活动',
  points: 3,
  description: `参加活动：${activityName}`,
}
```
